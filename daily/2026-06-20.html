CVE-2026-53519：哪吒监控严重路径遍历漏洞，一个GET请求拿下全站JWT密钥

CVSS 9.1 | 影响版本：< 2.0.13 | PoC已公开 | 无需认证 路径遍历

哪吒监控（Nezha Monitoring）被披露存在未授权路径遍历漏洞，攻击者无需任何认证，仅通过构造一个 GET 请求即可读取服务器上的敏感配置文件，获取 JWT 签名密钥，进而伪造管理员令牌接管整个面板。

漏洞原理：

  • 入口：构造 /dashboard../data/config.yaml
  • 绕过：strings.HasPrefix 前缀检查通过
  • 利用：filepath.Join 拼接路径逃逸出模板目录
  • 结果：读取 config.yaml 获取 JWT 密钥

攻击链：

  • 1. GET /dashboard../data/config.yaml → 获取 jwt_secret_key
  • 2. GET /dashboard../data/sqlite.db → 获取用户表、admin ID
  • 3. 用泄露的 HS256 密钥伪造管理员 JWT
  • 4. 以管理员身份登录，完全控制 Dashboard + 所有 Agent

值得关注：哪吒监控在国内自建服务器社区使用率极高，PoC已公开，攻击成本为零。建议立即升级到 v2.0.13+。
